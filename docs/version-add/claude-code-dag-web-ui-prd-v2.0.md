### 3.2 数据持久化 (P1)

#### 3.2.1 本地缓存策略

| 数据类型 | 存储方式 | 过期策略 |
|----------|----------|----------|
| 会话列表 | IndexedDB | 保留最近 100 条 |
| 会话详情 | IndexedDB | 保留最近 50 条 |
| 用户设置 | localStorage | 永久 |
| 临时状态 | Zustand (内存) | 会话结束清除 |

#### 3.2.2 缓存同步机制

```
1. 新会话创建 → 立即写入缓存
2. 会话更新 → 防抖写入 (500ms)
3. 页面刷新 → 优先读取缓存，再同步服务端
4. 存储空间不足 → FIFO 淘汰最旧记录
```

### 3.3 历史对话记录的存储和召回策略 (P1)

#### 3.3.1 存储策略

| 维度 | 策略说明 |
|------|----------|
| **存储结构** | 分层存储：会话 → 查询 (Query) → 工具调用链 |

| **分片存储** | 单个会话超过 10MB 自动分片 |

#### 3.3.2 召回策略

**1. 智能搜索召回**

| 搜索类型 | 实现方式 | 优先级 |
|----------|----------|--------|
| 关键词匹配 | 全文索引 (FlexSearch) | 高 |
| 语义搜索 | 本地 Embedding (Transformers.js) | 中 |
| 时间范围 | 时间轴过滤 | 高 |
| 标签过滤 | 标签索引匹配 | 高 |
| 工具类型 | 工具调用记录索引 | 中 |

**2. 上下文关联召回**

```
当用户发起新问题时，自动触发关联召回：

1. 关键词提取：从当前问题提取关键词
2. 历史匹配：在历史记录中搜索相似问题
3. 相关性排序：按关键词匹配度 + 时间衰减排序
4. 展示建议：在侧边栏展示 "相关历史记录"

示例：
用户问："如何优化 React 性能？"
→ 召回：3 天前 "React 渲染优化技巧"
→ 召回：1 周前 "useMemo 使用场景"
```

**3. 主动推荐召回**

| 场景 | 触发条件 | 推荐内容 |
|------|----------|----------|
| 新问题 | 检测到与历史问题相似度>0.8 | "你之前问过类似问题..." |
| 工具报错 | 检测到工具调用失败 | "上次遇到同样问题的解决方案" |
| 周期性任务 | 检测到周期性提问模式 | "这是你第 3 次问这个问题" |
| 学习路径 | 检测到连续学习行为 | "相关知识点：..." |

#### 3.3.3 召回算法

```python
# 伪代码：历史召回排序算法
def recall_history(current_query, history_records, top_k=5):
    scores = []
    
    for record in history_records:
        # 1. 关键词匹配分 (0-1)
        keyword_score = jaccard_similarity(
            extract_keywords(current_query),
            extract_keywords(record.question)
        )
        
        # 2. 语义相似度 (0-1) - 可选，使用本地模型
        semantic_score = cosine_similarity(
            embed(current_query),
            embed(record.question)
        )
        
        # 3. 时间衰减分 (0-1)
        time_decay = exp(-days_ago(record.createdAt) / 30)
        
        # 4. 使用频率分 (0-1)
        usage_score = min(record.view_count / 10, 1.0)
        
        # 加权总分
        total_score = (
            0.4 * keyword_score +
            0.3 * semantic_score +
            0.2 * time_decay +
            0.1 * usage_score
        )
        
        scores.append((record, total_score))
    
    # 按分数排序返回 top_k
    return sorted(scores, key=lambda x: x[1], reverse=True)[:top_k]
```
