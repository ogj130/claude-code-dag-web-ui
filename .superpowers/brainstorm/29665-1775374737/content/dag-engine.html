DAG 可视化渲染引擎

多 Agent 并行任务 DAG 的实时渲染方案

A

React Flow

成熟的 React 专用 DAG/流程图库，内置节点拖拽、自动布局、补间动画，支持自定义节点组件

  • ✅ React 原生，集成最顺滑
  • ✅ 内置 pan/zoom/拖拽，开箱即用
  • ✅ 社区活跃，文档完善
  • ⚠️ 包体积较大（~200KB）
  • ⚠️ 高度自定义时学习成本上升
B

ELK.js + 自定义 React 渲染

用 ELK.js 做自动布局算法，节点和边完全用 SVG/Canvas 自定义渲染，零依赖框架

  • ✅ 完全控制渲染细节
  • ✅ 包体积小（ELK 布局 + 自渲染）
  • ✅ 适合高度定制化风格
  • ⚠️ 需要自己实现交互（拖拽/缩放）
  • ⚠️ 开发工作量最大
C

D3.js + ELK 混合方案

D3 处理渲染和交互，ELK 做布局，各自做擅长的事

  • ✅ D3 灵活，DAG 渲染成熟
  • ✅ ELK 自动布局省心
  • ⚠️ D3 学习曲线较陡
  • ⚠️ 两个库结合有一定复杂度
D

Cytoscape.js

专业图可视化库，支持 DAG、网络图、流程图，布局算法丰富

  • ✅ 图算法最专业，布局算法最全
  • ✅ 支持 WebGL 渲染大图
  • ⚠️ 非 React 原生，需包装
  • ⚠️ UI 定制需要 CSS 层叠覆盖
