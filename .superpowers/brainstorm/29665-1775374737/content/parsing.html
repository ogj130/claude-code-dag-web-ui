Claude Code 输出解析方案

如何将 Claude Code CLI 的流式输出解析为结构化事件？

A

ANSI/Terminal 输出解析

直接解析 Claude Code 的 ANSI 彩色输出，通过特征字符串识别工具类型（[TOOL_CALL]、[AGENT] 等标记）

  • ✅ 无需修改 Claude Code 本身
  • ✅ 兼容所有 Claude Code 版本
  • ⚠️ 输出格式可能随版本变化，需要维护解析规则
  • ⚠️ 解析不够精确，可能误识别
B

Claude Code JSONL 日志模式

如果 Claude Code 支持 JSONL 输出格式（类似 --json 或 --log-format），直接解析结构化日志

  • ✅ 解析精确，零歧义
  • ✅ 未来版本兼容性好
  • ⚠️ 依赖 Claude Code 内置支持
  • ⚠️ 需调研 Claude Code 是否有此功能
C

MCP 协议 Hook 拦截

通过 MCP (Model Context Protocol) 协议的中间层拦截所有工具调用，获取完整的调用上下文

  • ✅ 最精确，获取完整调用上下文
  • ✅ 标准协议，不依赖 Claude Code 输出格式
  • ⚠️ 架构最复杂，需要 MCP 适配层
  • ⚠️ 与方案 A（本地直连）有架构冲突
