前端框架 & 实时通信选型

核心 UI 渲染栈与实时事件推送方案

A

Next.js + SSE (Server-Sent Events)

前后端同构，全栈 TypeScript，后端 route handlers 通过 SSE 推送工具/agent 执行事件，前端用 React 状态管理

  • ✅ 全栈统一技术栈，部署简单
  • ✅ SSE 实现简单，单向推送够用
  • ⚠️ Next.js 额外复杂度，对轻量项目略重
B

React + WebSocket

轻量 Node.js 后端 + React SPA，通过 WebSocket 全双工实时同步工具执行、任务状态、DAG 节点状态

  • ✅ 全双工通信，交互更灵活
  • ✅ 前端可反向推送（如取消任务）
  • ⚠️ WebSocket 需要心跳/重连处理
C

Vue 3 + SSE

Vue 3 Composition API 前端 + Node.js SSE 后端，更轻量的选择，适合快速原型

  • ✅ Vue 上手更快，模板语法简洁
  • ✅ 适合快速 MVP
  • ⚠️ 生态比 React 小，复杂场景可能受限
D

SolidJS + SSE

极致性能的响应式前端 + Node.js SSE，细粒度更新，适合高实时性 DAG 渲染

  • ✅ 性能最强，响应式精准更新
  • ✅ 对 DAG 节点频繁刷新有优势
  • ⚠️ 生态较小，学习曲线
