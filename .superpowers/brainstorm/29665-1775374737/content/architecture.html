CC Web — 平台架构选型

决定前后端通信方式和技术栈走向

A

本地 Agent 直连模式

Web 前端 + Node.js 后端进程管理，后端直接 spawn claude-code CLI 进程，通过 stdio/WebSocket 实时流式拉取输出

  • ✅ 部署最简单，数据不过公网
  • ✅ Claude Code 零改动兼容
  • ⚠️ 仅限本机使用，无多租户
B

MCP Proxy 中间层模式

保留 Claude Code CLI 独立运行，通过 MCP (Model Context Protocol) 协议代理工具调用，Web 实时订阅事件流

  • ✅ 可远程访问（SSH 隧道 / 内网穿透）
  • ✅ 架构解耦，扩展性强
  • ⚠️ 需要实现 MCP 协议适配层
C

完全独立 Web 模式

不依赖 Claude Code CLI，从零实现所有工具调用逻辑，Web 前端直连 API，通过 agent SDK 编排任务

  • ✅ 完全掌控，无 CLI 依赖
  • ✅ 可接入任意 LLM API
  • ⚠️ 工作量巨大，需重写所有工具实现
