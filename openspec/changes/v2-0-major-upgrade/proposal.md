## Why

当前 Claude Code DAG Web UI 的 v1.0 版本已经具备核心功能（DAG 可视化、终端输出、卡片系统），但存在以下关键问题：
1. **稳定性不足**：缺少 WebSocket 断连处理和错误边界机制
2. **数据易失**：无本地持久化，刷新页面数据丢失
3. **历史无法召回**：缺少历史对话的存储和智能检索能力
4. **性能瓶颈**：百节点级别 DAG 渲染可能卡顿
5. **交互体验单薄**：缺少快捷键、搜索、主题等增强功能

v2.0 旨在打造一个**稳定、持久、高性能、可扩展**的专业级 AI 工作流可视化工具。

## What Changes

### P0 - 稳定性提升
- **WebSocket 智能重连**：网络中断自动重连，支持指数退避（5s → 10s → 30s），最多 3 次重试，提供手动重连按钮
- **React 错误边界**：组件级错误隔离，单卡片崩溃不影响整体，提供"重新渲染"按钮
- **空状态处理**：无会话/无历史/加载中三种状态的友好 UI

### P1 - 数据持久化
- **IndexedDB 存储层**：基于 Dexie.js 实现，支持会话列表（最近 100 条）、会话详情（最近 50 条）的本地缓存
- **缓存同步机制**：新会话立即写入，更新防抖 500ms，页面刷新优先读缓存再同步服务端，存储空间不足时 FIFO 淘汰
- **数据压缩**：文本使用 LZ-String 压缩，大文件缩略图化，单会话超过 10MB 自动分片

### P1 - 历史对话存储与召回
- **分层存储模型**：Session → Query → ToolCall 层级结构
- **智能搜索召回**：关键词全文索引（FlexSearch）+ 语义搜索（Transformers.js）
- **上下文关联召回**：基于关键词相似度 + 时间衰减 + 使用频率的加权排序算法
- **主动推荐**：相似问题提示、错误解决方案推荐、周期性任务提醒
- **隐私与安全**：AES-256 加密、一键清除、JSON/Markdown 导出

### P1 - 性能优化
- **虚拟滚动**：只渲染可视区域节点，支持 1000+ 节点
- **节点分组**：同类型节点聚合显示，减少 50% 渲染量
- **懒加载**：折叠节点不渲染子节点，首屏加载 < 200ms
- **防抖重绘**：事件触发 100ms 后再更新
- **内存管理**：单会话最大节点数限制（默认 500），过期数据自动清理

### P2 - 交互体验增强
- **快捷键系统**：全局搜索（Cmd/Ctrl+K）、节点折叠/展开（Cmd/Ctrl+Shift+C/E）、主题切换、历史面板等
- **搜索功能**：会话历史、工具调用、问答内容的实时搜索，高亮匹配，支持时间范围和工具类型过滤
- **主题个性化**：暗黑/明亮/自动（跟随系统），6 种预设主题色，3 档节点密度，字体大小 12-18px 可调

### P2 - 监控与分析
- **Token 统计**：单次查询消耗、会话总消耗、日均趋势、成本估算（USD）
- **执行分析**：工具调用次数分布、平均响应时间、错误率趋势、热点工具排行

## Capabilities

### New Capabilities

**P0 稳定性相关：**
- `websocket-reconnect`: WebSocket 智能重连机制，支持指数退避和手动重连
- `error-boundary`: React 组件级错误边界，单卡片崩溃隔离
- `empty-states`: 空状态 UI 处理（无会话、无历史、加载中）

**P1 数据持久化相关：**
- `indexeddb-storage`: 基于 Dexie.js 的 IndexedDB 存储层
- `cache-sync`: 缓存同步机制（新会话立即写、更新防抖、页面刷新策略）
- `history-storage`: 分层存储模型（Session → Query → ToolCall，手动删除模式）
- `history-recall`: 智能召回策略（关键词搜索 + 上下文关联 + 主动推荐，v2.1 预留语义搜索）
- `data-privacy`: 数据隐私保护（AES-256 加密、一键清除、导出功能）

**P1 性能优化相关：**
- `virtual-scroll`: DAG 虚拟滚动，只渲染可视区域节点
- `node-grouping`: DAG 节点分组聚合
- `memory-management`: 内存管理（节点数限制、自动清理、大文件压缩）

**P2 交互体验相关：**
- `keyboard-shortcuts`: 全局快捷键系统
- `global-search`: 统一搜索功能（会话、工具、问答）
- `theme-system`: 主题系统（暗黑/明亮/自动 + 自定义）

**P2 监控分析相关：**
- `token-analytics`: Token 消耗统计和趋势分析
- `execution-analytics`: 执行分析（工具分布、响应时间、错误率）

## Impact

### 受影响代码模块
- `src/components/`: DAG 可视化组件（需增加虚拟滚动、节点分组）
- `src/stores/`: Zustand 状态管理（需增加持久化层）
- `src/hooks/`: WebSocket 相关 Hook（需增加重连逻辑）
- `src/utils/`: 工具函数（需增加 LZ-String 压缩、加密工具）

### 新增依赖
- `dexie` / `dexie-react-hooks`: IndexedDB 封装
- `flexsearch`: 全文搜索引擎
- `lz-string`: 文本压缩
- `crypto-js`: AES-256 加密
- `recharts`: 数据可视化图表

> **v2.1 预留依赖：** `@xenova/transformers`（语义搜索，可选）

### 系统架构变更
- 引入持久化层（IndexedDB），数据流从纯内存变为内存 + 磁盘双写
- DAG 渲染从直接渲染变为虚拟化渲染，需重构 ReactFlow 相关组件
- WebSocket 从简单连接变为带重连状态机的可靠连接

### 性能目标
- 百节点 DAG 渲染 < 500ms
- 首屏加载 < 2s
- 内存占用 < 200MB（正常会话）
- Lighthouse 性能分 > 90
